# ZRule — Open-Source Rule Engine

A self-hosted decision rule engine built with a visual JDM (JSON Decision Model) editor, multi-tenant organizations, and robust authentication.

Inspired by [GoRules BRMS](https://editor.gorules.io/) — fully open-source.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Runtime | [Bun](https://bun.sh) |
| Backend | [Hono](https://hono.dev) |
| ORM | [Drizzle ORM](https://orm.drizzle.team) |
| Database | PostgreSQL |
| Auth | [Better Auth](https://better-auth.com) (email/password + Organization plugin) |
| Frontend | React + Vite + [TanStack Router](https://tanstack.com/router) + [TanStack Query](https://tanstack.com/query) |
| Rule Editor | [@gorules/jdm-editor](https://github.com/gorules/jdm-editor) |
| Styling | Tailwind CSS + shadcn/ui |

## Features

### Rule Engine
- **Visual Decision Graphs** — drag-and-drop canvas for building rule flows
- **Decision Tables** — spreadsheet-style conditional logic (hit policies, unary operators)
- **Expression Nodes** — ZEN expression language for calculations
- **Function Nodes** — custom JavaScript with Monaco editor, TypeScript checking, built-in libraries (dayjs, big.js, zod)
- **Switch Nodes** — conditional branching
- **JSON Output** — produces standard JDM format, portable across GoRules SDKs
- **Version History** — every save creates a new version snapshot

### Authentication & Authorization
- **Email/Password** — sign up and sign in with email + password
- **Multi-tenant Organizations** — teams isolate their decision models
- **Role-based Access** — owner, admin, member roles per organization
- **Session Management** — secure HTTP-only cookies
- **API Scoping** — all endpoints are scoped to the user's organization memberships

### API
- `POST /api/auth/*` — Better Auth handler (signup, signin, session, org management)
- `GET /api/decisions` — list decisions (scoped to active org)
- `POST /api/decisions` — create a new decision model
- `GET /api/decisions/:id` — get a single decision with its version history
- `PUT /api/decisions/:id` — update the decision graph JSON
- `DELETE /api/decisions/:id` — soft-delete a decision
- `GET /api/organizations` — list user's organizations
- `POST /api/organizations` — create an organization

## Getting Started

### Prerequisites
- [Bun](https://bun.sh) (latest)
- [Docker](https://docker.com) (for PostgreSQL)
- Node.js 20+ (for some tooling)

### Setup

```bash
# 1. Clone and install
git clone <repo-url> && cd zrule
bun install

# 2. Start PostgreSQL
docker compose up -d

# 3. Configure environment
cp apps/server/.env.example apps/server/.env
# Edit .env with your values:
#   DATABASE_URL=postgresql://zrule:zrule@localhost:5432/zrule
#   BETTER_AUTH_SECRET=<openssl rand -base64 32>
#   BETTER_AUTH_URL=http://localhost:3001

# 4. Generate auth schema & run migrations
cd apps/server
bunx --bun auth@latest generate
bun run db:generate
bun run db:migrate

# 5. Start dev servers (from root)
bun run dev
```

The API runs on `http://localhost:3001` and the React frontend on `http://localhost:5173`.

### Available Scripts

| Command | Description |
|---------|-------------|
| `bun run dev` | Start both server and client |
| `bun run dev:server` | Start Hono server only |
| `bun run dev:client` | Start Vite dev server only |
| `bun run db:generate` | Generate Drizzle migrations |
| `bun run db:migrate` | Apply migrations to database |
| `bun run db:push` | Push schema directly (dev) |
| `bun run auth:sync` | Regenerate auth schema from config |
| `bun run build` | Build both packages |
| `bun run typecheck` | Type-check all packages |
| `bun run lint` | Lint all packages |

## Implementation Roadmap

### Phase 1 — Scaffold & Database
- Initialize Bun workspaces (`server` + `client`)
- Docker Compose for PostgreSQL
- Drizzle configuration and `bun:sql` client
- Decision models schema (`id`, `name`, `slug`, `org_id`, `graph` JSONB, `version`, timestamps)

### Phase 2 — Authentication (Better Auth)
- Install and configure Better Auth with Drizzle adapter (`provider: "pg"`)
- Run Better Auth CLI to generate auth schema (user, session, account, verification, organization, member, invitation tables)
- Enable `emailAndPassword` plugin
- Enable `organization` plugin for multi-tenancy
- Mount Better Auth handler on `/api/auth/*`
- CORS configuration for frontend origin
- Session middleware (stores `user`, `session`, `activeOrganization` in Hono context)

### Phase 3 — Rule Engine API
- Create decision CRUD routes, all scoped to the user's organization memberships
- Store JDM graph as `jsonb` in PostgreSQL
- Version tracking via `version` column or separate `decision_versions` table
- Zod validation for request bodies

### Phase 4 — React Frontend
- Vite + React + TanStack Router scaffolding
- Tailwind CSS + shadcn/ui setup
- Login/signup pages using Better Auth React client
- Organization switcher and management page
- Decision list dashboard with TanStack Query
- Decision editor page embedding `@gorules/jdm-editor`
  - `JdmConfigProvider` wrapper with WASM loading
  - `DecisionGraph` controlled component
  - Save/load wired to API via TanStack Query mutations

### Phase 5 — Polish
- Loading states, error boundaries, toast notifications
- `@hono/zod-validator` for request validation
- Optional: ZEN Engine WASM for in-browser simulation

## Key Implementation Details

### Better Auth Instance
```ts
// apps/server/src/auth/index.ts
import { betterAuth } from "better-auth";
import { drizzleAdapter } from "better-auth/adapters/drizzle";
import { db } from "../db";
import { organization } from "better-auth/plugins";

export const auth = betterAuth({
  database: drizzleAdapter(db, { provider: "pg" }),
  emailAndPassword: { enabled: true },
  plugins: [organization()],
  secret: process.env.BETTER_AUTH_SECRET,
  baseURL: process.env.BETTER_AUTH_URL,
});
```

### JDM Editor Integration
```tsx
// apps/client/src/routes/decisions.$id.tsx
import { DecisionGraph, JdmConfigProvider } from "@gorules/jdm-editor";
import "@gorules/jdm-editor/dist/style.css";

function DecisionEditor() {
  const [graph, setGraph] = useState(null);
  const { data: decision } = useQuery({
    queryKey: ["decision", id],
    queryFn: () => api.decisions[":id"].$get({ param: { id } }),
  });

  return (
    <JdmConfigProvider>
      <DecisionGraph
        value={graph ?? decision?.graph}
        onChange={(val) => setGraph(val)}
      />
    </JdmConfigProvider>
  );
}
```

### Auth Middleware
```ts
// apps/server/src/middleware/auth.ts
app.use("*", async (c, next) => {
  const session = await auth.api.getSession({ headers: c.req.raw.headers });
  c.set("user", session?.user ?? null);
  c.set("session", session?.session ?? null);
  c.set("activeOrganization", session?.session?.activeOrganization ?? null);
  await next();
});
```
