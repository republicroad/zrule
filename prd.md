## Product Overview

ZRule is an **open-source, self-hosted decision rule engine** with a visual JDM (JSON Decision Model) editor, multi-tenant organizations, and robust authentication. Inspired by [GoRules BRMS](https://editor.gorules.io/).

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Bun |
| Backend | Hono (port 3001) |
| ORM | Drizzle ORM |
| Database | PostgreSQL (via Podman) |
| Auth | Better Auth (email/password + Organization plugin) |
| Frontend | React + Vite + TanStack Router + TanStack Query (port 5173) |
| Rule Editor | @gorules/jdm-editor |
| Styling | Tailwind CSS + shadcn/ui |

---

## Core Features

### 1. Visual Rule Engine
- Drag-and-drop decision graph canvas
- Decision tables (spreadsheet-style, hit policies, unary operators)
- Expression nodes (ZEN expression language)
- Function nodes (custom JS, Monaco editor, TypeScript checking, built-in libs: dayjs, big.js, zod)
- Switch nodes (conditional branching)
- JSON output in portable JDM format
- Version history on every save

### 2. Authentication & Authorization
- Email/password sign up and sign in
- Multi-tenant organizations (team-level isolation of decision models)
- Role-based access: owner, admin, member per organization
- Secure HTTP-only cookie sessions
- API endpoints scoped to user's organization memberships

### 3. API

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/*` | Better Auth handler (signup, signin, session, org management) |
| GET | `/api/decisions` | List decisions (scoped to active org) |
| POST | `/api/decisions` | Create a new decision model |
| GET | `/api/decisions/:id` | Get a decision with version history |
| PUT | `/api/decisions/:id` | Update the decision graph JSON |
| DELETE | `/api/decisions/:id` | Soft-delete a decision |
| GET | `/api/organizations` | List user's organizations |
| POST | `/api/organizations` | Create an organization |

## Implementation Roadmap

### Phase 1 â€” Scaffold & Database
- Initialize Bun workspaces (server + client)
- Podman Compose for PostgreSQL
- Drizzle configuration and `bun:sql` client
- Decision models schema (id, name, slug, org_id, graph JSONB, version, timestamps)

### Phase 2 â€” Authentication (Better Auth)
- Better Auth with Drizzle adapter (`provider: "pg"`)
- Auth schema generation via CLI (user, session, account, verification, organization, member, invitation tables)
- `emailAndPassword` plugin
- `organization` plugin for multi-tenancy
- Mount auth handler on `/api/auth/*`
- CORS configuration
- Session middleware (user, session, activeOrganization in Hono context)

### Phase 3 â€” Rule Engine API
- Decision CRUD routes scoped to organization memberships
- JDM graph stored as JSONB in PostgreSQL
- Version tracking
- Zod validation for request bodies

### Phase 4 â€” React Frontend
- Vite + React + TanStack Router scaffolding
- Tailwind CSS + shadcn/ui setup
- Login/signup pages (Better Auth React client)
- Organization switcher and management page
- Decision list dashboard (TanStack Query)
- Decision editor page with `@gorules/jdm-editor` (JdmConfigProvider, DecisionGraph, save/load via API)

### Phase 5 â€” Polish
- Loading states, error boundaries, toast notifications
- `@hono/zod-validator` for request validation
- Optional: ZEN Engine WASM for in-browser simulation

## Non-Functional Requirements
- **Self-hosted**: deployable via Podman Compose
- **Open-source**: MIT license
- **Multi-tenant**: data isolation per organization
- **Type-safe end-to-end**: TypeScript, Drizzle ORM, Zod validation
